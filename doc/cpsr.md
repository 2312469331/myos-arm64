### G8.2.35
### 当前程序状态寄存器（CPSR）

#### 寄存器特性
##### 用途
保存处理器（PE）的状态与控制信息。

##### 配置条件
该寄存器仅在实现了 `FEAT_AA32`（AArch32 兼容特性）时存在。否则，直接访问 CPSR 的行为属于未定义。

##### 属性
CPSR 是一个 **32 位寄存器**。

---

### 字段说明
| 字段 | 位位置 | 名称/说明 |
|---|---|---|
| N | `[31]` | **负数条件标志**<br>由上一条设置标志位的指令的结果最高位决定。若将结果视为补码有符号整数，结果为负则置 1，为正或零则置 0。 |
| Z | `[30]` | **零条件标志**<br>上一条设置标志位的指令结果为 0 时置 1，否则置 0。常用于比较指令表示“相等”。 |
| C | `[29]` | **进位条件标志**<br>上一条设置标志位的指令产生进位/无符号溢出时置 1，例如加法的无符号溢出。 |
| V | `[28]` | **溢出条件标志**<br>上一条设置标志位的指令产生有符号溢出时置 1，例如加法的有符号溢出。 |
| Q | `[27]` | **累积饱和标志**<br>置 1 表示某些指令发生了溢出或饱和。 |
| - | `[26:24]` | 保留位，`RES0`（读为 0，写忽略）。 |
| SSBS | `[23]` | **推测存储旁路安全位（仅 FEAT_SSBS 实现时有效）**<br>禁止可能产生缓存时序侧信道的推测性加载/存储操作。<br>- `0b0`：禁止硬件以可能被侧信道利用的方式使用推测寄存器值。<br>- `0b1`：允许硬件在满足条件时以可能被侧信道利用的方式使用推测寄存器值。<br>异常进入非 Hyp 模式时，通常继承 `SCTLR.DSSBS` 的值；进入 Hyp 模式时，继承 `HSCTLR.DSSBS` 的值。热复位时，该字段为实现自定义值。未实现该特性时为 `RES0`。 |
| PAN | `[22]` | **特权访问禁止位（仅 FEAT_PAN 实现时有效）**<br>- `0b0`：翻译系统行为与 Armv8.0 一致。<br>- `0b1`：禁止特权态（EL1/EL2/EL3）对 EL0 可访问地址的读写。<br>异常进入时的行为：进入 EL1 且当前安全状态 `SCTLR.SPAN=0` 时置 1；进入 Secure EL3 且 `Secure SCTLR.SPAN=0` 时置 1；进入 Non-secure EL3 时强制置 0。未实现该特性时为 `RES0`。 |
| DIT | `[21]` | **数据无关时序位（仅 FEAT_DIT 实现时有效）**<br>- `0b0`：架构不保证指令执行时序与数据无关。<br>- `0b1`：要求数据无关时序的代码序列，执行时间必须与所有数据无关值无关。<br>热复位时，该字段置 0。未实现该特性时为 `RES0`。 |
| - | `[20]` | 保留位，`RES0`。 |
| GE | `[19:16]` | **大于等于标志**，用于并行加减运算。 |
| - | `[15:10]` | 保留位，`RES0`。 |
| E | `[9]` | **大小端状态位**<br>控制数据访问的加载/存储端序：<br>- `0b0`：小端模式<br>- `0b1`：大端模式<br>指令取指忽略该位。若硬件不支持大端，该位为 `RES0`；不支持小端则为 `RES1`。复位值继承 `SCTLR.EE` 配置。 |
| A | `[8]` | **SError 异常屏蔽位**<br>- `0b0`：不屏蔽异常<br>- `0b1`：屏蔽异常 |
| I | `[7]` | **IRQ 中断屏蔽位**<br>- `0b0`：不屏蔽中断<br>- `0b1`：屏蔽中断 |
| F | `[6]` | **FIQ 中断屏蔽位**<br>- `0b0`：不屏蔽中断<br>- `0b1`：屏蔽中断 |
| - | `[5]` | 保留位，`RES0`。 |
| - | `[4]` | 保留位，`RES1`。 |
| M | `[3:0]` | **当前处理器模式**<br>- `0b0000`：User<br>- `0b0001`：FIQ<br>- `0b0010`：IRQ<br>- `0b0011`：Supervisor<br>- `0b0110`：Monitor<br>- `0b0111`：Abort<br>- `0b1010`：Hyp<br>- `0b1011`：Undefined<br>- `0b1111`：System |

---

### CPSR 的访问方式
- 读取：使用 `MRS` 指令。
- 写入：使用 `MSR (register)` 或 `MSR (immediate)` 指令。
