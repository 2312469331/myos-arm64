# D1.5  处理器状态（PSTATE）
处理器状态（PSTATE）是**处理器状态信息**的抽象统称。

所有 Arm 指令集架构，都提供可操作 PSTATE 各字段的专用指令。

PSTATE 包含以下全部字段：
- 仅在 AArch32 状态下有效的字段；
- 仅在 AArch64 状态下有效的字段；
- 在两种执行状态下均有效的字段。

在架构伪代码中，PSTATE 被定义为 `ProcState` 类型的 **PSTATE 结构体**。

---

## D1.5.1  AArch64 专属 PSTATE 字段
AArch64 状态下有效的 PSTATE 字段，分为以下类别：
1. 条件标志位
2. 执行状态控制位
3. 异常屏蔽位
4. 访问控制位
5. 时序控制位
6. 推测执行控制位

### D1.5.1.1  PSTATE 字段的访问方式
在 AArch64 状态下，通过**特殊系统寄存器**访问 PSTATE：
- 使用 `MRS` 指令直接读取；
- 使用 `MSR` 指令直接写入。

#### 系统寄存器与 PSTATE 字段映射
| 专用系统寄存器 | 关联 PSTATE 字段 |
|--------------|----------------|
| NZCV | N、Z、C、V |
| DAIF | D、A、I、F |
| CurrentEL | EL |
| SPSel | SP |
| PAN | PAN |
| UAO | UAO |
| DIT | DIT |
| SSBS | SSBS |
| TCO | TCO |
| ALLINT | ALLINT |
| PM | PM |
| SVCR | SM、ZA |

其余 PSTATE 字段**无直接读写通道**。

软件可通过 **`MSR（立即数）`** 指令，直接改写以下 PSTATE 成员：
`D、A、I、F、SP、PAN、UAO、DIT、SSBS、TCO、ALLINT、PM、SM、ZA`

#### 异常等级访问权限
1. EL0 可直接访问：
`PSTATE.N/Z/C/V、SSBS、DIT、TCO、SM、ZA`
2. 满足条件时，EL0 可访问 `PSTATE.D/A/I/F`：
`SCTLR_EL1.UMA=1` 且 `HCR_EL2.{E2H,TGE}≠{1,1}`
3. `SM、ZA` 访问指令：EL0 及更高等级均可执行
4. 其余 PSTATE 访问指令：仅 EL1 及以上可用，**EL0 执行未定义**

#### PSTATE 写操作的副作用约束
对 PSTATE 字段的写入会影响处理器运行行为，且满足以下硬性规则：
- 该写入产生的副作用，**不会**对流水线中更早执行的指令生效；
- 该写入产生的副作用，**必定**对后续所有指令生效；
其余附加硬件副作用不做强制保证。
