# D1.4.1.6 异常向量

当异常捕获到**AArch64 模式**的异常等级时，CPU 从异常向量地址开始执行代码。

异常向量地址处的内存，需要存放对应异常类型的处理函数。

某异常等级 ELx 的向量基地址，由**向量基地址寄存器** **`VBAR_ELx`** 定义。
每一类异常，都相对于向量基地址，拥有一个固定偏移的专属向量入口。

在 AArch64 架构下，为分配对应的异常向量，会根据以下条件对异常进行分类：

- 异常归属类型：
  — 普通同步异常（**不含同步外部终止异常**）
  — 同步外部终止异常
  — 系统错误异常（SError），包含虚拟 SError、委派 SError 异常
  — 普通/虚拟 IRQ 中断
  — 普通/虚拟 FIQ 中断
- 异常触发时的源异常等级
- 当前使用的栈指针（SP）类型
- 寄存器组运行状态

同步异常参考：同步异常类型章节；异步异常参考：异步异常类型章节。

下表列出：**叠加在向量基地址之上**、用于确定最终异常向量入口的固定偏移量。

### EASE 位有效取值规则

- 异常捕获到 EL1：使用 `SCTLR2_EL1.EASE` 有效位
- 异常捕获到 EL2：使用 `SCTLR2_EL2.EASE` 有效位
- 异常捕获到 EL3：使用 `SCR_EL3.EASE` 有效位

针对上表补充规则：
若目标异常等级为 EL3，则 EL3 的直接下级异常等级判定规则：

1. 若 EL2 已实现、且在异常源安全域下已启用 → 下级为 EL2
2. 若 EL2 未实现 / 未启用 → 下级为 EL1

# 重点解答你最大的疑问：

## 两张表偏移看着重复，**完全不冲突**

1. **表1 管辖**
   普通报错、缺页、SVC 系统调用、IRQ 外设中断、FIQ 快速中断
2. **表2 管辖**
   总线报错、外设访问失败、硬件严重错误（SError、外部内存/设备异常）

✅ 同一偏移地址，**只会触发其中一类异常**：

- 0x000
  普通指令报错 → 查表1
  硬件总线访问崩了 → 查表2
  分类不同、不会同时触发，所以地址重叠完全合法。

## 先固定 GICv2 计算公式（QEMU virt 平台通用）

- 第1个单元格：
  - `0x00` = SPI 全局外设中断 → (\boldsymbol{GIC\_IRQ = 32 + 第二个值})
  - `0x01` = PPI 私有核内中断 → (\boldsymbol{GIC\_IRQ = 16 + 第二个值})

***

## 一、PPI 核内中断（CPU 私有）

### 1. 通用定时器 timer

```dts
interrupts = <0x01 0x0d 0x104  0x01 0x0e 0x104  0x01 0x0b 0x104  0x01 0x0a 0x104>;
```

- `0x01 0x0d` → (16+13 = \boldsymbol{29}) 👉 **EL1 物理定时器（你要用）**
- `0x01 0x0e` → (16+14 = \boldsymbol{30}) 安全物理定时器
- `0x01 0x0b` → (16+11 = \boldsymbol{27}) 虚拟定时器
- `0x01 0x0a` → (16+10 = \boldsymbol{26}) EL2 虚拟化定时器

### 2. PMU 性能计数器

```dts
interrupts = <0x01 0x07 0x104>;
```

(16+7 = \boldsymbol{23})

### 3. GIC 内部中断

```dts
intc@8000000: interrupts = <0x01 0x09 0x04>;
```

(16+9 = \boldsymbol{25})

***

## 二、SPI 外设中断（32 + 二号值）

### 1. PL011 串口 UART

- `pl011@9000000`  `<0x00 0x01 0x04>` → (32+1 = \boldsymbol{33})
- `pl011@9040000`  `<0x00 0x08 0x04>` → (32+8 = \boldsymbol{40})

### 2. PL061 GPIO

- `pl061@9030000`  `<0x00 0x07 0x04>` → (32+7 = \boldsymbol{39})
- `pl061@90b0000`  `<0x00 0x00 0x04>` → (32+0 = \boldsymbol{32})

### 3. PL031 实时时钟

`pl031@9010000` `<0x00 0x02 0x04>` → (32+2 = \boldsymbol{34})

### 4. 全部 VirtIO-MMIO 设备

统一规则：
`<0x00 0x10 0x01>` \~ `<0x00 0x2f 0x01>`

- 从 (32+10 = \boldsymbol{42}) ～ (32+47 = \boldsymbol{79}) 连续排列

| 设备                    | DTS 中断号 | GIC 真实号 |
| --------------------- | ------- | ------- |
| virtio\_mmio\@a000000 | 0x10    | 42      |
| virtio\_mmio\@a000200 | 0x11    | 43      |
| ...                   | ...     | ...     |
| virtio\_mmio\@a003e00 | 0x2f    | 79      |

# ARM GICv2 中断号 终极算法（配合你设备树 + A53 官方标准）

先记死**GICv2 全局编号规则**，一步到位，再也不会算错。

***

## 一、GICv2 中断分区（固定硬件规定）

### 1. SGI 软件中断 0～15

`IRQ 0 ~ 15`
核间中断、软件触发，一般不用来做外设/定时器

### 2. PPI 私有外设中断 16～31

`IRQ 16 ~ 31`
**每个CPU核心独占**，定时器、CPU 私有硬件全在这
👉 你 Timer 全是 PPI

### 3. SPI 全局外设中断 32～...

`IRQ 32 起步`
UART、GPIO、网卡、控制器……所有全局外设

***

## 二、核心公式（最重要）

### PPI 真实 GIC 中断号：

\[
\mathbf{GIC\_IRQ = 16 + 设备树里的第二个值}
]

***

## 三、套你 timer 设备树，现场演算

你设备树：

```
0x01 0x0a 0x104
0x01 0x0b 0x104
0x01 0x0d 0x104
0x01 0x0e 0x104
```

每组：`<PPI标记(1)  偏移号  配置>`

- (16 + 0x0a(10) = \boldsymbol{26}) → EL2 Hypervisor Timer
- (16 + 0x0b(11) = \boldsymbol{27}) → 虚拟 Timer
- (16 + 0x0d(13) = \boldsymbol{29}) → ✅ **EL1 非安全物理定时器（你要用）**
- (16 + 0x0e(14) = \boldsymbol{30}) → 安全世界 Secure Timer

***

## 四、SPI 怎么算？（顺带学会）

设备树第一字段是 `0` 代表 SPI：

```
<0  xx  0x104>
```

公式：
\[
\mathbf{GIC\_IRQ = 32 + xx}
]

***

## 五、极简背诵版

1. 设备树第一个数 = `1` → PPI
   \[
   irq = 16 + 第二个数
   ]
2. 设备树第一个数 = `0` → SPI
   \[
   irq = 32 + 第二个数
   ]
3. SGI 固定 0～15
